Корневой URL:    api/v1/

Функционал для администратора системы:

  • авторизация в системе (регистрация не нужна (польз: "api", пароль: "123"))
    через Login API-Admin: ./api-auth/login/

  • Корнень для администратора ./api-admin/
    Все ссылки для опросов, вопросов и ответов на этой странице.

  • добавление/изменение/удаление опросов.

    survey/ - список опросов
    survey/<int> - просмотр, изменений, удаление опроса

    Атрибуты опроса: название, дата старта, дата окончания, описание.
    После создания поле "дата старта" у опроса менять нельзя -???

  • добавление/изменение/удаление вопросов в опросе.

    question/ - список вопросов
    question/<int> - просмотр, изменений, удаление вопроса

    Атрибуты вопросов: текст вопроса, тип вопроса (ответ текстом,
    ответ с выбором одного варианта, ответ с выбором нескольких вариантов)

  • Cтандартные методы GET, PUT, POST

Функционал для пользователей системы:

  • получение списка активных опросов: ./surveys/

  • прохождение опроса: опросы можно проходить анонимно, в качестве идентификатора пользователя в API передаётся числовой ID, по которому сохраняются ответы пользователя на вопросы; один пользователь может участвовать в любом количестве опросов

    Просмотр и прохождение опроса по ./survey/<номер_ID_опроса>/ например по ./survey/2/ (survey без S)

    Если ответ один или несколько, то убирается ненужные словари из списка ответов.
    Если "Свой ответ", то записывается текстовое сообщение без списка и словарей.

  • получение пройденных пользователем опросов с детализацией по ответам (что выбрано) по ID уникальному пользователя

    пока не реализовано до конца, но после прохождения опроса можно увидеть вопросы и свои ответы.

Использовать следующие технологии: Django 2.2.10, Django REST framework.
